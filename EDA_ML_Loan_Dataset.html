    • ChgOffDate: There are 736465 missing values which accounts for 81.9% of total records. It contains date on which loan is declared to be default.
    • RevLineCr: There are 4528 missing values. It should contains two unique values indicating whether the type of loan is revolving line of credit or not.
    • LowDoc: There are 2582 missing values. It should contains two unique values indicating whether it is LowDoc loan program or not.

    • DisbursementDate: There are 2368 missing values. It contain dates on which loan amount got disbursed.

    • MIS_Status:: There are 1997 missing values. It indicates the status of loan.

    • Bank:: There are 1559 missing values.

    • BankKState: There are 1566 missing values.

    • NewExist:There are 136 missing values.It indicates whethere the loan is applied for new business or existing business.

    • City, nameand state contains 30,14 and 14 missing values respectively.
  • ApprovalFY denotes the year on which approval was given. Since it denotes year, it has to be a numerical column. But currently it of object type.
  • RevLineCr and LowDoc are binomial variables. But it contains 18 and 8 unique values respectively. There should be only two unique values.
  • ApprovalDate and DisbursementDate are of object type. But, it has to be date type.
  • CreateJob and RetainedJob are highly correlated. So , RetainedJob should be ignored for analysis

Things to do before proceeding to analysis

  • ChgOffDate has very high number of missing values and also we are not worried about the loan charged off date. So, It is better to remove the column instead of imputation.
  • RevLineCr has some absurd and missing values. Those records containing values other than 'Y' and 'N' and also missing values could be removed.
  • LowDoc has some absurd and missing values. As a standard practise, we know that lowdoc program is given if the loan amount is less than $1500000. Based on this, we can impute values for this column.
  • DisbursementDate, MIS_Status, Bank, BankKState, NewExist, City, name, state contains very less number of missing values. So, the records containing missing values in these columns could be removed.
  • DisbursementGross, BalanceGross, ChgOffPrinGr, GrAppv, SBA_Appv contains symbols and special characters. We need to remove those characters to convert it to numeric columns
  • ApprovalDate and DisbursementDate should be converted into date type.
  • Replacing the values of NewExist with 0 and 1.
  • Creating new column 'is_franchise' from 'FranchiseCode' column.
